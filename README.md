Clickable Prototype — Design Explanation
CodeAlpha UI/UX Design Internship — Task 4
1. Overview
BiteRush is a food delivery mobile application designed for Android. This document explains the clickable prototype built in Figma, which connects the high-fidelity screens created in the earlier task into a single interactive user flow. The prototype demonstrates how a user moves from opening the app to placing an order, simulating a real food-ordering experience.
2. Objective of the Prototype
The goal of this prototype is to validate the usability of the BiteRush interface by linking individual static screens into a working sequence of interactions. It allows a reviewer to click through the app exactly as an end user would, testing navigation logic, button placement, and overall flow before any actual development begins.
3. Screen Flow
The prototype follows a linear, easy-to-follow path that mirrors a typical food ordering journey. Each screen connects to the next through a tap interaction on Figma's prototyping canvas.

4. Interaction & Prototype Linking
All navigation uses simple Tap interactions with Instant transitions, keeping the flow fast and predictable.
Primary action buttons (Login, Add to Cart, Checkout) are styled in a consistent orange to draw the user's eye and signal the next step clearly.
The flow is designed as a closed loop — Splash → Login → Home → Product Detail → Cart → Thank You → back to Home — so a user can keep ordering without ever hitting a dead end.
Frames are organised on a single Android Medium board in Figma, making the prototype easy to present and test in one continuous view.
5. Design Rationale
The colour palette pairs a deep navy with a warm orange accent, giving the app an appetising, energetic feel that suits a food delivery brand. Bold typography and large tappable buttons were chosen to keep the interface beginner-friendly and accessible on mobile screens. Each screen carries only the information needed for that step of the journey, reducing cognitive load and keeping the ordering process quick and frustration-free.
6. Conclusion
This clickable prototype brings together all of BiteRush's individual screens into one cohesive, testable experience. It demonstrates a complete and realistic food-ordering flow, from opening the app to receiving an order confirmation, and forms the final deliverable of the CodeAlpha UI/UX Design Internship.
